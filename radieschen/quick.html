Formatting Help

The wiki markup used by Radieschen has been compiled from the best markup ideas used in other wiki systems. No surprises here. Please note that the html renderering is done by Swing's standard JEditorPane widget. This is a limitation espcially when displaying complex external WWW-pages.

Headers 

= Level 1 header = 
== Level 2 Header ==
=== Level 3 Header ===
==== Level 4 header ====
===== Level 5 Header =====

gives:

Level 1 Header

Level 2 Header

Level 3 Header

Level 4 Header

Level 5 Header

Please note that the = must be at the start of the line; there must be a space between the last = and the header text and there must be a line separator after the terminating = at the end of the header.

Paragraphs

This is a paragraph. Please note that a paragraph MUST start at the first character of a line. Otherwise it might be treated as the continuation to a preceeding list item.

This is a second paragraph. Paragraphs are separated by empty lines.

Text between two lines starting with :: appears centered. For example: 

::
This is centered 
::

will give:

This is centered

Text between two lines starting with -- will be treated as a quote. For example: 

--
 This is a quote from an ancient master of wisdom
--

will yield:

This is a quote from an ancient master of wisdom

Text Embellishments

  • Enclosing text in '' or ** makes the text bold. Example: ''fat'' gives fat.
  • Enclosing text in ~~ puts text into italics. Example: ~~italy~~ makes italy.
  • Enclosing text in __ puts text in subscript. Example __sub__script yields subscript.
  • Enclosing text in ^ puts text in superscript. Example: ^super^script makes superscript.
  • Coloured text can be produced by enclosing the text into ~~colourname: and ~~. Example: ~~red:This is red~~ yields: This is red Only the standard html colours are supported.

Forced line breaks within a paragraph can be obtained by giving %%%.

Links

  • A link is created by enclosing the URL in []. Example: [http://www.psi.ch] yields http://www.psi.ch. All URL's not starting with either http or ftp are treated as wiki internal links. Page creation happens by creating a new wiki link in some page.
  • Optionally the link may be given a differerent name then the URL. The the URL must be followed by a | and the name within the []. Example: [http://www.psi.ch|PSI] gives PSI.
  • images can be included by enclosing their URL in [[ ]]; double square brackets. This looks like [[path-to-image]].

Lists

Nesting differerent list types is not supported. Extending the items text on another line is supported, but the text must not start at the first character of the line.

Itemized Lists 

* noeff
** naff
* niff

gives

  • noeff
    • naff
  • niff

The * character must be at the start of the line; the item text must be separated by a space. Deeper nesting is expressed by giving more *.

Ordered Lists 

# noeff
## naff
# niff

gives

  1. noeff
    1. naff
  2. niff

The # character must be at the start of the line; the item text must be separated by a space. Deeper nesting is expressed by giving more #.

Definition Lists 

;niff: niff text
;noeff: noeff text

yields

niff
niff text
noeff
noeff text

The ; must be at the start of the line; The definiton text follows after the :. Nesting definition lists is not allowed.

Tables

Simple tables can be done with ||. Example: 

|| t1 || t2 || t3 ||
|| t4 || t5 || t6 ||
|||span 2 || t7||

creates

t1 t2 t3
t4 t5 t6
span 2 t7

Please note the spaces around the table elements. The table row must be terminated with a return after the ||. Elements spanning multiple columns can be created by adding additional | to the column separator ||.

Preformatted Text and Escapes

Single characters can be escaped by preceeding them with a backslash. This is useful when using those sysmbols which are special to wiki markup in text. Text enclosed in {{{ }}} is treated as preformatted text.

Script generated Content

It is possible to use the beanshell scripting language within Radieschen files in order to generate content dynamically. This works similar to JSP. The follwoing syntax is supported:

<% script %>
execute script and insert output. The output must be formatted as Radieschen markup.
<%=var %>
insert value of variable named var.
<%! script%>
execute script and print nothing.
